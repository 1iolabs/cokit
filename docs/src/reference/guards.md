# Guards
Guards are permission checks for transactions. Transactions without [permissions] will be rejected.

### Checks
#### Is Participant
The simplest permission check is that you have to be a participant in order to write new transactions to the [CO](/reference/co.md)

#### POA
The Proof-of-Authority Consensus mechanism used in COs. Transactions are validated by approved accounts, known as validators. Validators run software allowing them to put transactions in blocks. The process is automated and does not require validators to be constantly monitoring their computers.
